# RFC 9457

RFC 9457 is the current Problem Details standard for HTTP APIs. It obsoletes RFC 7807.

Use media type:

```http
application/problem+json
```

Baseline standard fields:

- `type`: URI reference identifying the problem type.
- `title`: short human-readable summary of the problem type.
- `status`: HTTP status code generated by the origin server.
- `detail`: human-readable explanation specific to this occurrence.
- `instance`: URI reference identifying this occurrence.

Common extension fields:

- `code`: stable machine-readable error code.
- `request_id`: request/error correlation ID.
- `errors`: field-level validation errors.

Important compatibility note:

- RFC 7807 is the older Problem Details RFC.
- RFC 9457 is the newer standard and should be treated as the target contract.

Implementation rule:

```text
RFC 9457 fields + stable product extensions = final API error contract
```
