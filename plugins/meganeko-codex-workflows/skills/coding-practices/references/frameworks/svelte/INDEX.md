# Svelte/SvelteKit Framework

Use this branch for Svelte and SvelteKit components, routing, layouts, load functions, forms, progressive enhancement, SSR/CSR boundaries, stores, runes, and UI packages.

## Architecture Routing

Apply policy in this order:

1. the base coding-practices gates;
2. the [frontend area index](../../areas/frontend/INDEX.md) and every focused frontend reference selected by the changed concerns;
3. the [TypeScript index](../../languages/typescript/INDEX.md) and selected TypeScript references when TypeScript is active; and
4. [Svelte Project Structure And Lifecycle Mapping](./project-structure.md) for SvelteKit files, runtime placement, and lifecycle mechanisms.

This index is a router. Frontend references own architecture and behavior policy, TypeScript references own type-system mechanics, and the project-structure reference maps those decisions onto Svelte/SvelteKit constructs without redefining them.
